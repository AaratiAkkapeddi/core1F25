Project 7: Toggle

Due 11/11 9am

For this assignment, you'll create an interactive webpage that gives users control over their experience through clicking. Using the classList toggle pattern we learned in class, design a feature where users can customize some aspect of your page's appearance or content by clicking a button or element. This customization should align with your harmonic collection theme. For example, if your theme is furniture design, users might toggle between modern and vintage styles; if it's desserts, they might switch between 'chocolate' and 'vanilla' modes, or show/hide nutritional information; if it's music, they might click to activate a CSS animation that rotates a 'record' image or switch color schemes for different genres. The key is that clicking should give the user a meaningful choice that enhances their interaction with your themed content. Your page should include at least one button or clickable element and at least one element that visually changes when the toggle is activated.

Example of classList toggle pattern:

                let button = document.getElementbyId("color-button");

                let colorSwatch = document.getElementbyId("color-swatch");

                button.addEventListener("click", function(){       

                    if(colorSwatch.classList.contains("pink")){   
     
                        colorSwatch.classList.remove("pink");

                    }else{         

                        colorSwatch.classList.add('pink');    
 
                    }

                });

Technical Requirements:

  • Figma design with four frames -- before clicking/after clicking for both desktop and mobile.(submit link in canvas)
  • Use @media queries to make sure your website works on both desktop and mobile
  • At least 1 element that is clickable
  • At least 1 element that changes on clickable

Grading Criteria:

  • Conceptual depth (30%): Clear, intentional perspective shift that engages with device context
  • Design execution (30%): Both versions are polished, cohesive, and well-designed
  • Technical implementation (30%): Clean code, effective use of media queries, JS, semantic HTML
  • Integration with portfolio theme (10%): Successfully connects to your larger body of work
