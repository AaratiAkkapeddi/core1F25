Project 6: Time as Material

Due 11/4 9am

Create a single-page website that uses CSS keyframes to animate elements related to your harmonic collection theme over time. The animations should tell a visual story or demonstrate transformation, with interactive hover effects.

Learning Objectives:

  • Master CSS keyframes and animation properties
  • Practice using the :hover pseudo-class for interactive animations
  • Practice proper HTML structure and semantic markup
  • Understand external CSS file linking (no more css in your html files!)
  • Use classes and IDs appropriately
  • Organize files in a logical folder structure

Technical Requirements:

  • Figma design for both desktop and mobile. (submit link in canvas)
  • Use @media queries to make sure your website works on both desktop and mobile
  • At least 1 @keyframes animations
  • At least 1 element that animates on :hover (can be combined with keyframe animation or separate from)

Theme-Based Examples:

  • If your theme is desserts: A cake that gradually gets frosted (layers appear one by one) revealing the different components that make up the cake.
  • If your theme is music: Sound waves that pulse continuously to a beat
  • If your theme is furniture: Chair that assembles piece by piece

Suggested Animation Patterns:

  • Entrance animations - Elements fade in or slide in when page loads
  • Continuous loops - Elements that rotate, pulse, or float indefinitely
  • Sequential animations - Using animation-delay to create a choreographed sequence
  • Hover animations (REQUIRED) - Elements that animate when cursor hovers over them
  • Transformation timeline - Show a process over time (making coffee, building furniture, etc.)

Examples of :hover Animations

Hover triggers keyframe animation:

            .element:hover {

                animation: spin 2s ease-in-out;

            }

Hover on parent affects child:

            .container:hover .child {

                animation: bounce 1s infinite;

            }

Grading Criteria:

  • Conceptual depth (30%): Clear, intentional perspective shift that engages with device context
  • Design execution (30%): Both versions are polished, cohesive, and well-designed
  • Technical implementation (30%): Clean code, effective use of media queries, semantic HTML
  • Integration with portfolio theme (10%): Successfully connects to your larger body of work
