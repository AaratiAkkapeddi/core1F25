Project 5: Shifting Perspectives

First, read Shannon Matternâ€™s article in Avant. Design and code a single page that presents two different perspectives of the same content, using CSS media queries to create distinct experiences for desktop and mobile viewing. This assignment explores how context shapes perception and how responsive design can be used not just for layout adaptation, but for conceptual storytelling.

Concept:

Consider how we experience things differently depending on our context. A phone is intimate, personal, held close. A computer screen is often more associated with work and viewed from more of a distance. Your design should reflect these different viewing contexts and how they might change the way we understand or interact with your content.

Technical Requirements

  • One HTML file
  • CSS using at least one media query (minimum breakpoint at 768px) Significantly different layouts/presentations between mobile and desktop views Both versions must be fully functional and intentionally designed (not just "responsive by default")

Conceptual Requirements:

Your design should explore perspective shift in a way that relates to your harmonic collection theme.

Consider:

  • What changes between contexts? (hierarchy, emphasis, information revealed/hidden, tone, mood)
  • How does viewing distance/intimacy affect the experience?
  • What's appropriate for each device context?

Quick Example Conceptual Approaches:

  • Desserts: Desktop content reflects a communal meal at a full table, vs on mobile it's more about solo eating showing a recipe for just one.
  • Furniture: Desktop content reflects furniture for shared public space, vs on mobile it's more about furniture for the intimate space of the home.

Your design must include a desktop and mobile version. ***Also submit a brief written explanation (2-4 sentences) describing: Your conceptual approach to the perspective shift, How the mobile vs desktop contexts informed your design decisions, Technical challenges you solved.

Evaluation Criteria:

  • Conceptual depth (30%): Clear, intentional perspective shift that engages with device context
  • Design execution (30%): Both versions are polished, cohesive, and well-designed
  • Technical implementation (30%): Clean code, effective use of media queries, semantic HTML
  • Integration with portfolio theme (10%): Successfully connects to your larger body of work
